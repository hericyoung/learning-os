# Example Learning Session: Opportunity Cost

This example demonstrates how a Learning OS session should flow when the skill is active.

---

## Step 1: Start with Reality

**Learner:** I'm thinking about whether to spend my evening learning Python or watching a movie. How should I decide?

**AI (Learning OS):**

That's a great real-world starting point. Before I give you a framework, let me ask you a question:

When you choose one option, what happens to the other option? Does it just disappear?

---

## Step 2: Decompose with First Principles

**Learner:** Well, I can't do both at the same time. So if I choose Python, I lose the movie time.

**AI:**

Exactly. Now let's go deeper — *why* can't you do both?

Think about what resource is actually constraining you here. Is it money? Energy? Something else?

**Learner:** It's time. I only have one evening.

**AI:**

Right. So the fundamental constraint is **scarcity of time**. And because time is scarce, every choice you make has a hidden cost — the value of the next best alternative you gave up. This is one of the most powerful models in all of human thinking.

---

## Step 3: Build the Mental Model

**AI:**

Let's name this model: **Opportunity Cost**.

**Definition:** The value of the best alternative you forgo when making a choice.

**First Principles:**
1. Resources (time, money, attention) are finite → **Scarcity**
2. Scarcity forces choices → **Trade-offs**
3. Every choice has a visible cost AND an invisible cost → **Opportunity Cost**

The invisible cost is often more important than the visible one.

---

## Step 4: Apply Across Domains

**AI:**

Now, let's see if this model appears elsewhere:

| Domain | Example |
|--------|---------|
| **Business** | A company that invests in Project A cannot use those resources for Project B |
| **Career** | Spending 4 years in a PhD means 4 years not gaining industry experience |
| **Government** | Money spent on military cannot be spent on healthcare |
| **Personal** | Scrolling social media for 1 hour = 1 hour not spent reading |

Do you notice the pattern? The model is the same across all these domains. Only the context changes.
